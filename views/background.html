Project background

This project originated after the conversations with the teacher Enric Mayol. Enric introduced to us the FamilySearch organization and the existence of their API that granted access to genealogical data.

FamilySearch is a non profit organization that aims to connect families through their generations. Their vision as a collective is to help people create links with their ancestors so they can better understand who they are, create a family bond and build the bridge between the past and the future.

This project is constructed around the API that FamilySearch created to grant access to their genealogical data. The project consists of three main blocs or sections:

  • In-deep analysis of the API. The objective of this section is to understand what data is truly available through the API and how the different components are connected to each other.
  • In-deep analysis of the API. The objective of this section is to understand what data is truly available through the API and how the different components are connected to each other.
  • Implementation of some examples that interact with the API.

Project context and motivations

During college I felt a higher empathy for those projects and subjects that required certain levels of abstraction. The process of understanding a given problem, design a valid solution and finally execute it, always felt more rewarding than the deep technological details.

On top of that, I always defined myself as a jack of all trades, therefore I was eager to find a project that fell out of my knowledge or comfort area, so I could learn something new and break the routine.

For this two reasons the proposed project by Enric Mayol felt like something I could enjoy, given the fact that until that point I knew almost nothing about genealogy or web coding.

This project had another particularity. The fact that the quality of the project outcome would completely rely on the ability to properly understand how to link the accessible data to doubts or questions that society might have, made this project both scary and interesting at the same time.

Finally, by the time I enrolled myself with this project, I was working as a user experience intern. This project felt like a good opportunity to carry some of the learned knowledge into a project.
